## なぜ「推し活」なのか

さて、なぜ私が『推し活駆動アウトプット』というスタンスでいるのか？というと、「推し活は好きや敬意の塊であり、敬意によって生み出されるアウトプットには、心地の良い熱量が灯るから」です。  
ポジティブな熱量をもとにアウトプットをしていると、もっと、この技術の良さ、この経験を伝えたい！という原動力が生まれます。この原動力は、発信者に「同じ気持ちの仲間を増やすために、できる限り読み手や聞き手にわかりやすく届けるには、どうしたらよいか？」を強く意識させます。その意識を元に改善を繰り返すことで、結果的として必要な人に届きやすいアウトプットが生まれ、発信者が愛するジャンルのファンを増やすことにつながります。  
    
自らが愛するジャンルのファンが増えること。いわゆる「布教成功」という状態ですね。同じジャンルを好む人が増えれば、アウトプットの総量も増えるはずです。また、そうして増えていった仲間のアウトプットによって、自分が救われることもあるでしょう。  
  
だからこそ、アウトプットしたいテーマやジャンルを『推し』にすることで、発信者も周囲もポジティブな気持ちでアウトプットに関わることができるのです。
  
## 具体的な行動
推し活駆動アウトプットのメリットについて確認したところで、具体的な行動の例も紹介させてください。  
これは私個人が意識している行動の例ではありますが、
  
* 何よりも、アウトプットのテーマや技術ジャンル、発信する場を好きになる
* 「この技術のここが好き！」「この技術のおかげで助かった！」を、具体的なエピソードを添えて発信する
* どうしても自分の力でアウトプットする気力が湧かない時は、他の仲間のアウトプットの良いところや好きなところを探して言語化してみて、熱量を再確認する
* 勉強会やイベントへのポジティブな参加レポートを執筆・公開し、発信や交流をするための場をつくる人へのリスペクトを表明する
  
などです。  
色々書きましたが、上記の行動例に縛られる必要はありません。情報の受け手が『愛』を感じ取ることができる状態であれば、推し活駆動アウトプットは大成功だと言って良いでしょう。  
  
## 気をつけたいこと
推し活駆動アウトプットを行うにあたり、気をつけるべきことが1つだけあります。  
それは、特定の技術やジャンルが好きということを盾に、他のものを貶めるような発信をしていないか気を配ることです。これは、特定のものへの愛にあふれる発信者がやってしまいがちな行動です。
例えば、ツール A を褒めたい一心で、類似の機能をもつツール B と比較し、

```text
Bは機能の割に値段が高くてダメなので、Aがオススメ
```
などと自分のブログに書いてしまう、というシチュエーションが該当します。特に機能の比較を主題とした発信では、自らが推したいツールの優位性をアピールするために、悪気なくこの表現をしてしまっているアウトプットを見かけます。  
  
「ダメ」という強い否定の言葉は、いくら悪気がなくても、ツール B を『推し』ている人を不快にしてしまいかねません。その結果、あなたのブログを読んだツール B 推しの人は、ツール A を嫌いになってしまう可能性もあります。せっかくツール A が好きな仲間が増えるような活動をしているのに、自分の行動で将来は仲間になってくれるかもしれない人を失うのは、もったいないですよね。  

これを回避するには、それぞれのサービスにおいて優れた点を抽出して記載をし、可能な限り具体的なユースケースを示した言い換えにすることをおすすめします。例えば、以下のような書き方です。

``` text
ツールAの特徴は、なんと言っても圧倒的なコストの安さです。手軽に導入ができるため、仕組みそのものの導入につなげるための検証や、学習目的での利用にちょうど良いです。
一方、ツールBの優位性はその安定性にあります。SLA 99.9%を保証しており、接続数がYY以上の大規模な商用環境への導入に向いています。
```

あなたにとっての推しがあるように、あなたが比較対象として取り上げたものを推している人もいます。ネガティブな要素で相手のマイナス面に注目させなくても、あなたの推しの素晴らしさを表現する方法はあるはずです。ぜひ、気を配ってみてください。
